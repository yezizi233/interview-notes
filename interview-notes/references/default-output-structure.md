# Default Output Structure

Use this reference only when the user did not provide a finished template.

The default structure is derived from the approved sample interview-notes document. Ignore the sample cover page. Follow the body structure only.

## Shape

The body should usually follow this pattern:

1. Title line naming the interview subject
2. Major section 1
3. Major section 2
4. Major section 3
5. Within each major section, one or more bracketed topic headers
6. Under each topic header, layered bullets

## Chinese Pattern

Typical Chinese form:

- `访谈对象X：<类别>——<名称>`
- `一、第一部分：……`
- `【主题】`
- first-level bullet for main point
- second-level bullet for support, conditions, exceptions, examples

## English Pattern

When the output language is English and no English template exists, preserve the same logic rather than the exact Chinese markers:

- `Interview Subject X: <type> — <name>`
- `1. Part One: ...`
- `[Topic]`
- first-level bullet for main point
- second-level bullet for support, caveat, comparison, exception

## Bullet Logic

Use first-level bullets for:

- topic statements
- summary judgments
- key findings
- category labels that introduce a cluster of details

Use second-level bullets for:

- supporting detail
- conditions and exceptions
- examples
- quoted support
- sub-comparisons

Do not flatten a two-level structure into one long bullet list when the source clearly supports hierarchy.

## Tables

Use a table when a topic becomes numerically dense or comparison-heavy.

Examples:

- multiple vendors compared across dimensions
- lists of rankings
- dates and phases
- numeric ranges or percentages

Keep explanatory bullets below the table when necessary.

## Quotes

Use direct quotes selectively:

- when wording materially affects meaning
- when a judgment is stronger if preserved in the speaker's own language
- when a ranking or caveat needs evidentiary support

Do not overquote. The clean file is still a notes document, not a transcript.
